# 研究与发现（执行版 2026-05-20）

## 当前结论

当前最适合的变现路径不是同时做很多 AI 产品，而是：

```text
n8n + Dify 自动化工作流
-> 真实客户案例
-> 企业知识库
-> 私有化部署 / Agent 定制
```

简历工具和电商文案工具已经部署，继续作为引流资产和能力背书。Etsy、SillyTavern、声音克隆、短视频内容工厂短期不做主线，后续有资源或客户需求再评估。

## 主线工具调研

### n8n

- **仓库**：https://github.com/n8n-io/n8n
- **用途**：自动化流程核心，连接表格、邮箱、飞书、企微、API、数据库和 AI 节点。
- **部署方式**：
  ```bash
  docker run -d --name n8n -p 5678:5678 n8nio/n8n
  ```
- **商业注意**：Sustainable Use License，自托管使用通常可行，但如果包装成云服务转售，需要仔细看条款。
- **当前变现场景**：
  - 客服自动回复
  - Excel / CSV 自动处理
  - 日报周报自动生成
  - 内容批量生成与半自动发布

### Dify

- **仓库**：https://github.com/langgenius/dify
- **用途**：LLM 应用开发、工作流、RAG、Agent 编排。
- **部署方式**：
  ```bash
  git clone https://github.com/langgenius/dify
  cd dify/docker
  docker compose up -d
  ```
- **商业注意**：社区版适合起步，复杂多租户和企业管理能力需要谨慎评估。
- **当前变现场景**：
  - AI 工作流节点
  - 企业知识库 Demo
  - 客服 / HR / 运营助手

### DeepSeek / Claude API

- **定位**：
  - DeepSeek：低成本，适合文案、分类、总结、批量处理。
  - Claude：复杂分析、长文档理解、关键客户交付时使用。
- **策略**：默认低成本模型，复杂任务再切换高质量模型。

### Qdrant

- **仓库**：https://github.com/qdrant/qdrant
- **用途**：向量数据库，企业知识库检索层。
- **当前价值**：Month 2 做知识库时使用，提升可控性和私有化交付能力。

### Docling

- **仓库**：https://github.com/DS4SD/docling
- **优势**：IBM 出品，适合 PDF、Word、PPT、图片 OCR 等文档解析，结构保留较好。
- **商业用途**：MIT 协议，可商用。
- **当前价值**：企业知识库文档预处理优先选择。

### RAGFlow

- **仓库**：https://github.com/infiniflow/ragflow
- **用途**：RAG 引擎和知识库应用，适合做企业知识库参考方案。
- **当前价值**：Month 2 和 Dify / Docling 对比，选最稳定的交付路径。

### LiteLLM

- **仓库**：https://github.com/BerriAI/litellm
- **用途**：统一多个模型 API，方便切换 DeepSeek、Claude、OpenAI、本地模型。
- **当前价值**：Month 2 后接入，用来做成本控制和模型路由。

### Langfuse

- **仓库**：https://github.com/langfuse/langfuse
- **用途**：LLM 调用追踪、成本监控、效果分析。
- **当前价值**：有客户后接入，方便定位问题和证明服务质量。

## 后置工具调研

### Open WebUI

- **仓库**：https://github.com/open-webui/open-webui
- **用途**：自托管 AI Chat 界面。
- **当前定位**：Month 4+ 私有 AI 工作台，不作为 Week 0-2 重点。

### AnythingLLM

- **用途**：私有 AI 工作区、RAG 和团队知识库。
- **当前定位**：私有化方案备选，和 Open WebUI 对比。

### LocalAI / Ollama

- **用途**：本地模型运行，适合数据不能出内网的客户。
- **当前定位**：Month 4+ 私有化部署使用。

### ComfyUI

- **仓库**：https://github.com/comfyanonymous/ComfyUI
- **用途**：节点式图像生成工作流。
- **商业注意**：GPL 协议，作为 SaaS 或深度集成时需注意合规。
- **当前定位**：短视频内容工厂 / 电商图生成后置，不占用 Month 1-2 主线。

### faster-whisper

- **仓库**：https://github.com/SYSTRAN/faster-whisper
- **用途**：音视频转写、字幕生成。
- **商业用途**：MIT 协议，可商用。
- **当前定位**：短视频内容工厂或会议转写客户出现后启用。

### MegaTTS3 / CosyVoice

- **用途**：中文 TTS、配音、语音生成。
- **商业注意**：MegaTTS3 需确认商用 License；CosyVoice 可作为备选。
- **当前定位**：配音和短视频方向后置。

### RVC-WebUI

- **仓库**：https://github.com/RVC-Project/Retrieval-based-Voice-Conversion-WebUI
- **用途**：声音克隆 / 音色转换。
- **风险**：必须获得声音授权，避免侵权。
- **当前定位**：暂缓，不做前 3 个月主线。

### Marker

- **仓库**：https://github.com/VikParuchuri/marker
- **用途**：PDF 转 Markdown，适合 RAG 预处理。
- **商业注意**：GPL-3.0，商用需谨慎，必要时购买商业授权。
- **备选方案**：Docling、MinerU。
- **当前定位**：只作为文档解析备选，不单独做 SaaS。

### SillyTavern

- **仓库**：https://github.com/SillyTavern/SillyTavern
- **用途**：AI 角色扮演 / 陪伴。
- **商业注意**：AGPL-3.0，且有内容安全、支付和社区治理风险。
- **当前定位**：暂不执行。

## 关键发现

1. **接单先于产品**：现阶段最缺的是真实需求、案例和信任，不是更多功能。
2. **n8n 是最大杠杆**：客户能理解“省重复工作”，比理解“Agent”容易得多。
3. **Dify 适合作为 AI 能力中台**：可以同时支撑工作流和后续知识库。
4. **两个现有工具不能放弃，但不能重投入**：它们负责引流、验证付费意愿和做能力背书。
5. **免费项目必须有边界**：1 个流程、2 周交付、1 次修改，换案例和评价。
6. **私聊是转化动作，不是流量来源**：先通过内容、服务帖和工具获得互动，再私聊。
7. **Etsy / 配音 / 声音克隆不是当前主线**：可以作为资料保留，但前 2 个月不投入。
8. **Month 1 的真正目标是案例**：收入重要，但 3 个可展示案例更能支撑后续提价。
