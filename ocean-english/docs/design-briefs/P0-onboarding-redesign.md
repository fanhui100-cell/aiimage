---

## 1. 这是什么屏（现状）
新用户首次进入的定级 onboarding，路由 `app/onboarding`，组件 `components/screens/OnboardingScreen.tsx`。当前 4 大步：

1. **goal 目标**：6 张考试卡（雅思 / 托福 / 六级 / 四级 / 考研 / 休闲）
2. **path 学习方式**：4 条（全面掌握 / 速记词汇 / 精读提升 / 应试备考）
3. **channel 定级方式**：快速测评（二分认词）｜我知道我的水平
4. **test/selfcheck**（认词二分 + 微调）**或 pick**（7 张等级卡直选）→ **result 结果页**（等级卡 + CEFR + 词量 + 每日目标 8/12/16/20/自定义 → 开始学习）

顶部有进度条（5 段）+「跳过设置」。

---

## 2. P0 要达成的 4 件事
1. **每步可返回上一步**（当前只能前进）。
2. **学习方式(path) 真正落地**（当前选了不保存、无效）。
3. **目标 ↔ 七等级星系对齐**（双轴：能力档 × 目标考试讲清楚）。
4. **结果页「为你定制」摘要**（起点星系 → 目标星系 的成长路径 + 每日量 + 路径 + 预计周期）。

---

## 3. 功能改动清单（改 / 加 / 删）

### ➕ 新增
- **返回控件**：每步左上「← 上一步」**且**顶部进度条每段可点击回跳到该步；返回保留已选状态。
- **结果页「为你定制」摘要卡**：
  - 成长路径可视化：`起点星系 →（中间星系…）→ 目标星系`（用等级带的金色轨道隐喻）。
  - 摘要行：目标考试 · 当前能力档(CEFR) · 每日 N 词 · 学习路径 · 预计周期（天/周，按 词量÷每日量 粗算）。
- **双轴说明**：在 goal 或 result 处用一句话/小图说明「**能力档 = 你现在在哪（初中→SAT 七档）**；**目标 = 你想去哪（考试）**」。

### ✏️ 修改
- **Step1 目标格**：补 **SAT**；明确「目标=目的地」。建议目标项 = 四级/六级/考研/托福/雅思/SAT/休闲（初中、高中是**起点能力**不作为目标，由定级得出）。每个目标需能映射到一个星系（见 §4 表）。
- **进度条**：现 5 段写死，改为能正确反映「双通道」（测评 / 直选 两条等价为第 4 段）且可点。
- **result 等级卡**：词量口径统一到星系真实词数（见 §4 数据注意）。

### ❌ 删除
- 无硬删除。「跳过设置」保留，但跳过 = 落默认档（建议 L3 四级）并提示「可在『我的』页重新定级」。

## 4. 数据契约（字段 —— 屏幕的输入与产出）

### 屏幕最终要产出（交给我写进 store `Profile`）：
| 字段 | 类型 | 说明 |
|---|---|---|
| `targetExam` | string | 目标考试 id（见下表）|
| `path` | `'full'｜'words'｜'reading'｜'exam'` | **学习方式（新增持久化）** |
| `level` | 1–7 | 能力档（定级结果）|
| `band` | number | = min(level+1, 8)，派生 |
| `dailyGoal` | number | 每日新词（8/12/16/20/自定义 4–60）|

### 7 个能力档（等级带星系，来自 `lib/levels.ts`）
| level | 档 | 星系 id | CEFR |
|---|---|---|---|
| 1 | 初中 | junior | A2 |
| 2 | 高中 | senior | B1 |
| 3 | 四级 | cet4 | B1 |
| 4 | 六级 | cet6 | B2 |
| 5 | 考研 | kaoyan | B2 |
| 6 | 托福 | toefl | C1 |
| 7 | SAT | sat | C1 |

### 目标考试 → 星系映射（成长路径终点）
| 目标 | 星系 | 类型 |
|---|---|---|
| 四级 CET-4 | cet4 | 等级带 |
| 六级 CET-6 | cet6 | 等级带 |
| 考研 KaoYan | kaoyan | 等级带 |
| 托福 TOEFL | toefl | 等级带 |
| SAT | sat | 等级带 |
| 雅思 IELTS | ielts | 主题星系 |
| 休闲 Casual | mine（我的星云）/ 无强目标 | — |

> **路径算法**：起点 = level 对应等级带星系；目标在等级带上 → 画「起点 →…→ 目标」沿金色轨道、标跨越几座；目标是主题星系(雅思) → 画「起点 → ✦ 雅思主题星系」。
> **词量口径注意**（给我，不影响设计）：`lib/levels.ts` 现用「源表条数」(初中3223…)，宇宙星系卡用「去重词数」(初中1987…)。result 词量请用**星系去重词数**，我接入时统一。

## 5. 各步骤与状态（设计要覆盖）
对每步给出：默认 / hover / 选中 / 返回态；移动端单列。
- **goal**：目标卡网格（含 SAT）。选中高亮。
- **path**：4 条卡 + 图标 + 描述。**这一步的选择必须在 result 摘要里体现**（呼应"path 落地"）。
- **channel**：两入口（快速测评 / 我知道我的水平）。
- **test**：认词卡（词 + CEFR 标签）+「认识 / 不认识」+ 轮次 (n/6)。
- **selfcheck**：微调认词（词 + 中文）。
- **pick**：7 张等级卡（档名 + 描述 + 词量），与 §4 对齐。
- **result**：能力档大卡（档名 / Level n · CEFR / 词量）+ **成长路径可视化** + **定制摘要行** + 每日目标选择（8/12/16/20/自定义）+「开始学习 →」。
- **全局**：顶部 brand「词渊 Lexiverse」+ 可点进度条 + 「← 上一步」+ 「跳过设置」。
