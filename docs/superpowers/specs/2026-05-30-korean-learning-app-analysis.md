# 韩语学习套件 完整技术分析文档

> 参考原型：@进击的柯基怪（抖音）日语版
> 分析日期：2026-05-30
> 目标：以日语版为蓝本，复刻为韩语版独立 Web + 桌面 App 产品
> 语言方向调整理由：市场更大、汉字词优势更强、技术难度更低、竞品几乎空白

## 一、项目全貌

三个相互关联的韩语学习工具，定位"替代 Anki 的沉浸式韩语学习系统"：

| 工具 | 名称（韩语版） | 核心亮点 |
|------|-------------|---------|
| 1 | 우주 별자리 어휘도（宇宙星座词库） | 词汇以星球呈现，한자어词根聚成星座 |
| 2 | TOPIK 기출문제 시스템（TOPIK 真题系统） | 导入任意 TOPIK PDF，AI 解析题目结构 |
| 3 | 한국어 PDF 독해기（韩语 PDF 阅读器） | 韩语原文生词自动标注词义 |

**为什么韩语比日语更适合这个产品：**
- 中国学韩语人数 > 学日语，且增速更快（韩剧/K-pop 驱动）
- 60-70% 的韩语词汇是 **한자어（汉字词）**，中国用户天然有词源联想优势
- 韩语只有一套书写系统（한글），无竖排文字，OCR 难度显著低于日语
- TOPIK 历年真题在 topik.go.kr 全部免费公开，数据获取比 JLPT 容易
- 现有竞品几乎空白，Anki 替代方向几乎无人深耕

## 二、系统一：우주 별자리 어휘도（宇宙词库）

### 2.1 视觉层完整拆解

**核心设计决策：双层架构**

```
3D 星系模式  ──  探索 / 沉浸 / 传播 / 视觉核心卖点
2D 学习模式  ──  日常查词 / 复习 / 做题 / PDF 工作流
```

3D 是灵魂，2D 是生产力。两种模式共用同一份词库数据，随时可切换。纯 3D 视觉震撼但难以每天高效使用；学习行为需要稳定的平面界面承载。

**3D 星系模式（主视觉，Three.js 实现）**
- 真实 3D 空间，节点有 x/y/z 三维坐标（离线预计算后存库）
- 节点 = 发光彩色球体，语义类别决定颜色，`InstancedMesh` 单次 draw call 渲染万级节点
- 节点间细线 = 语义关联 / 汉字词根金色轨道线（独立可切换层）
- 星云粒子背景（`THREE.Points` + custom shader）
- 后期处理：Bloom 发光 + Depth of Field 景深（`@react-three/postprocessing`）
- `CameraControls`：旋转/缩放/平移，点击节点后相机**平滑飞行**过去
- 文字标签：`CSS2DRenderer`（跟随 3D 节点但 DOM 渲染，清晰不模糊）

**2D 学习模式（高效学习，PixiJS 或 graphology + SVG 实现）**
- 浅色泡泡模式：白/灰背景，圆形气泡节点 + 半透明 category bubble
- 深色平面模式：黑背景 + 发光节点，2D 力导向布局
- 类别标签：감정 / 사회 / 자연 / 일상 / 학문 / 한자어군

**语法关系图（Grammar Map）**
- 展示韩语语法项之间的关系
- 核心语法类别：
  - TENSE（시제：과거/현재/미래）
  - ASPECT（상：진행/완료/반복）
  - MODALITY（양태：추측/의무/가능）
  - SPEECH LEVEL（경어법：반말/존댓말/격식체）
  - CONDITION（조건：-면/-다면/-아야）
  - CAUSE（이유：-아서/-으니까/-기 때문에）
  - CONCESSION（양보：-아도/-더라도）
  - CONJUNCTION（연결：-고/-지만/-는데）

**动词/形容词变形展开图（Conjugation Wheel）**
- 中心：原形（e.g. 먹다 / 가다）
- 外围放射状展开所有变形：
  - 먹어요 / 먹었어요 / 먹을 거예요（존댓말）
  - 먹어 / 먹었어 / 먹을 거야（반말）
  - 먹고 / 먹지만 / 먹으면 / 먹어서（연결형）
  - 먹을 수 있다 / 먹고 싶다 / 먹어야 해（보조 동사）
  - 먹히다（피동）/ 먹이다（사동）⚠️ 见下方说明

**특별 기능：한자어 词根展开图**
- 韩语独有功能，日语版没有
- 点击한자어节点，展开所有同词根词汇
- 例：경(경제, 경험, 경쟁, 경우) → 全部在星图上高亮连线
- 对中国用户极其有用（直接关联中文汉字）

### 2.2 词卡数据结构（韩语版）

每个单词一张结构化词卡：

```
词卡示例：눈물（눈물）
────────────────────────────────
MEANING
  中文：眼泪，泪水
  English：tear (drop from the eye)

PRONUNCIATION
  한글：눈물
  발음：[눈물] (nunmul)
  国际音标：/nun.mul/

WORD TYPE（词汇分类）
  고유어（固有词）← 韩语原生词汇，非汉字来源

TOPIK LEVEL：TOPIK I / 2급

SCENARIO（使用场景）
  表达感动、悲伤、痛苦等情绪流出的泪水
  · 졸업식에서 친구와 헤어지며 눈물을 흘렸다
  · 슬픈 영화를 보고 눈물이 났다

ETYMOLOGY（词源）
  고유어 분류：固有词，纯韩语词根
  눈（眼睛）+ 물（水）→ 眼睛里的水 → 眼泪
  ※ 눈물 是日常最常用表达；泪水的汉字词写法为 누루(淚淚) 等，实际韩语中极少单独使用

MNEMONIC（汉字词对比记忆 / 场景故事）
  一段100-150字中文叙事，结尾高亮词的发音
  韩语版特殊优势：可以利用中韩汉字联系做记忆桥接

EXAMPLES（例句）
  · 눈물이 나다（流泪）—— 动词搭配
  · 눈물을 흘리다（流下眼泪）—— 动词搭配
  · 눈물을 참다（忍住泪水）—— 动词搭配
  每句含：韩语原文 + 罗马字标注 + 中文翻译 + TOPIK级别

RELATIONS
  同义/相关：울다（哭）/ 슬프다（悲伤）/ 감동（感动）/ 눈시울（眼眶）
  反义词：웃음（笑）/ 웃다（笑）
  上位词：분비물（分泌物）
  搭配词：눈물이 나다 / 눈물을 흘리다 / 눈물을 참다 / 눈물을 닦다
```

**한자어词卡示例（最核心差异化功能）**：

```
词卡：경제（경제）
────────────────────────────────
MEANING
  中文：经济
  English：economy, economics

WORD TYPE：한자어（漢字語）

HANJA（汉字）：經濟（경제）
  경(經)：经，经营、经历、经过
  제(濟)：济，救济、帮助

CHINESE CONNECTION（汉字关联 —— 对中国用户专属）
  中文：经济（jīng jì）
  发音对比：경제[경제] ≈ 경(jīng)제(jì)
  意义完全相同：98%语义一致

RELATED 한자어（同词根）
  경(經)：경험(经验) / 경쟁(竞争) / 경로(路线)
  제(濟)：구제(救济) / 경제(经济) / 제도(制度)

TOPIK LEVEL：TOPIK II / 4급
```

```

```

### 4.2 韩语版 OCR 优势

韩语 PDF 比日语 PDF 容易处理：
- **只有一套文字（한글）**，无汉字/假名混排
- **无竖排文本**，始终横排
- TOPIK 真题 PDF 全部为数字版（非扫描），可直接提取文字
- 仅部分旧书/漫画需要 OCR

## 五、技术难点完整分析

### 难点 1：3D 星图渲染性能 ⭐⭐⭐⭐⭐

**问题**：TOPIK 词库约 7000-12000 词，3D 场景中全量渲染会严重卡顿

**解决方案：Three.js + React Three Fiber**

目标是真实 3D 星系（有景深、轨道线、粒子），必须用 Three.js，不能用 PixiJS 2D 替代。

**性能关键策略**：

| 问题 | 解决方案 |
|------|---------|
| 万级节点渲染 | `InstancedMesh`：所有节点共用一个 mesh，单次 draw call，GPU 并行渲染 |
| 点击命中检测慢 | Raycasting + 空间八叉树（`three-mesh-bvh`），仅对可见节点检测 |
| 文字标签在 3D 里模糊 | `CSS2DRenderer`（DOM 元素跟随 3D 坐标，始终清晰）或 SDF text |
| 相机迷失方向 | 点击节点 → `CameraControls.fitToBox()` 平滑飞行过去 |
| 3D 布局计算慢 | **离线预计算** x/y/z 坐标并存库，启动时直接读取，不实时计算 |
| 远处节点细节 | LOD：相机距离 > 阈值时节点缩小/隐藏标签，只显示集群中心 |
| 视觉效果 | Bloom + DOF（`@react-three/postprocessing`），粒子背景（`THREE.Points`）|

**graphology** 仍然负责图数据管理，只是布局改用 `d3-force-3d`（D3 力导向的 3D 版本）预计算坐标。

**韩语版额外层**：한자어词根金色轨道线作为独立 `LineSegments` 对象，可随时切换显示。

### 难点 2：语义聚类算法 ⭐⭐⭐⭐⭐

**问题**：如何自动将韩语词汇聚类成语义群，同时识别汉字词群

**解决方案（Pipeline）**：
```
1. 多语言向量嵌入
   → paraphrase-multilingual-mpnet-base-v2（支持韩语）
   → 输入：한글单词 + 中文义 + English义 拼接

2. UMAP 降维（高维→2D）
   → 保留语义邻近关系，相似词自然聚集

3. HDBSCAN 密度聚类
   → 自动发现集群数量

4. Claude API 为集群命名
   → 输入集群内前20个词 → 输出：감정/사회/자연/일상/학문...

5. 额外：한자어词根聚类
   → 按汉字词首字提取词根
   → 将同词根词标记为 hanja_group
   → 在星图上用特殊颜色连线显示词根关系

6. 结果写入数据库（map_x, map_y, cluster_id, hanja_root）
```

**韩语独有优势**：汉字词根聚类是本系统对中国用户的核心差异化，日语版没有这个层次。

### 难点 3：韩语形态分析（NLP Pipeline）⭐⭐⭐

**问题**：韩语是黏着语，词语边界比日语更复杂，同一词根有数十种变形

**工具链**：
```python
# 服务端（Python）
konlpy                →  韩语 NLP 框架（包含多个分析器）
konlpy.tag.Okt        →  快速分词 + 词性标注（推荐 MVP）
konlpy.tag.Mecab      →  高精度（需要 MeCab-ko 安装）
korean_romanizer      →  韩语→罗马字转写（발음 标注）
jamo                  →  한글 字母拆解（자모 분해）
hanja                 →  汉字转换（고유어/한자어 识别）

# 客户端（浏览器）
hangul.js             →  韩文字母拆解（类似 wanakana）
```

**韩语 vs 日语 NLP 复杂度对比**：
- 日语：需要3套字符系统（平/片/汉）识别
- 韩语：只有한글，但黏着性更强（一个词干+无数词尾）
- 韩语分词比日语更容易（字符系统简单），但变形识别更复杂

### 难点 4：韩语词形变化规则引擎 ⭐⭐⭐⭐

**问题**：韩语动词/形容词变形比日语更复杂，有7种不规则变化类型

**韩语变形分类**：

```python
# 规则变形（规칙 변용）
가다 → 가요 / 갔어요 / 가면 / 가서 / 가도 / 가야 해요
먹다 → 먹어요 / 먹었어요 / 먹으면 / 먹어서

# 불규칙 변용（7种不规则）
ㅂ불규칙: 덥다 → 더워요（ㅂ→우）
ㄷ불규칙: 듣다 → 들어요（ㄷ→ㄹ）
ㅅ불규칙: 낫다 → 나아요（ㅅ탈락）
ㅎ불규칙: 파랗다 → 파래요（ㅎ탈락+변형）
ㄹ불규칙: 살다 → 사세요/사는（ㄹ탈락）
르불규칙: 모르다 → 몰라요（르→ㄹ+아/어）
우불규칙: 푸다 → 퍼요（우탈락）

# 변형 종류（约40种形式）
forms = {
  "해요체_현재": "가요",         # 存在体现在
  "해요체_과거": "갔어요",       # 存在体过去
  "해요체_미래": "갈 거예요",    # 存在体未来
  "반말_현재": "가",
  "반말_과거": "갔어",
  "연결_고": "가고",             # 并列
  "연결_지만": "가지만",         # 转折
  "연결_서": "가서",             # 原因
  "연결_면": "가면",             # 条件
  "연결_도": "가도",             # 让步
  "보조_싶다": "가고 싶다",      # 想要
  "보조_수있다": "갈 수 있다",   # 能够
  "보조_아야하다": "가야 해",    # 必须
  "피동": "가지다",              # 被动
  "사동": "가게 하다",           # 使动
  "관형사형_현재": "가는",       # 现在时定语形
  "관형사형_과거": "간",         # 过去时定语形
  "관형사형_미래": "갈",         # 未来时定语形
  # ...
}
```

**实现策略**：
- 规则引擎（不靠 AI）：先判断词干类型（규칙/불규칙），再应用变形规则
- 库：`korean-verb-conjugator`（Python）或自实现规则引擎
- 所有变形实时计算，不存储

### 难点 5：TOPIK PDF 结构解析 ⭐⭐⭐

**问题**：TOPIK 试卷 PDF 格式每届略有差异

**好消息**：
- TOPIK PDF 全部为数字版，无需 OCR
- 格式比 JLPT 更一致（官方标准化程度高）
- PyMuPDF 直接提取文字质量高

**解决方案**：
```
1. PyMuPDF 提取文字和坐标
2. 识别题号（숫자. 패턴）
3. 识别选项（①②③④）
4. Claude API 做语义级结构解析
5. 结果缓存：同一 PDF hash 不重复解析
```

### 难点 6：韩语 OCR（次要）⭐⭐

**问题**：仅부분 PDF（旧书/漫画/手写）需要 OCR

**解决方案**：
- **PaddleOCR**（韩语准确率高于 EasyOCR）
- 或保留本机已安装的 EasyOCR（支持韩语，够用）
- 韩语横排单一字符系统，OCR 难度远低于日语

### 难点 7：词卡数据批量生成 ⭐⭐

**韩语版特殊处理**：

## 十一、总结

**完全可行，且比日语版更适合做成商业产品**：

| | 日语版 | 韩语版 |
|--|--------|--------|
| 市场规模 | 中 | 大 |
| 竞品情况 | 有（Anki/Wanikani/BunPro） | 几乎无 |
| 最大技术难点 | 星图性能 + 3套文字 | 星图性能 + 变形规则 |
| 核心差异化 | 宇宙视觉 | 宇宙视觉 + **한자어 星座** |
| 最大风险 | JLPT 数据获取 | 例句质量（AI 补充） |

**最大风险是韩语例句质量**：Tatoeba 韩语语料比日语少，需要 Claude API 批量生成高质量例句，要做 prompt 调优和人工抽检。
