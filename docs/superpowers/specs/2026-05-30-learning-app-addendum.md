## 二、技术路线统一说明

### PixiJS vs Three.js —— 最终定案

**结论：Three.js + React Three Fiber 是主渲染方案，不使用 PixiJS。**

用户明确要求"真实 3D 星系模式"，这是产品的核心差异化卖点。两者不是质量优劣之分，而是目标不同：

| | PixiJS | Three.js |
|--|--------|----------|
| 适合场景 | 2D/2.5D 高性能图谱 | 真实 3D 星系（景深/粒子/轨道/相机飞行）|
| 本项目选择 | 不采用 | ✅ 主渲染方案 |

2D 学习模式使用 CSS/SVG/Canvas 2D 实现，不引入 PixiJS。

## 三、MVP 验收标准

### 3.1 Phase 0：3D 星系视觉原型

| 指标 | 目标 |
|------|------|
| 节点数量 | 1000-2000 mock 节点 |
| 帧率 | 稳定 60fps（Chrome，中端 GPU）|
| 点击响应 | 点击节点 → 相机飞行动画 < 800ms 完成 |
| 标签 LOD | 远景无标签，近景显示选中节点标签，无卡顿 |
| 视觉目标 | 截图给不了解项目的人看，能感受到"宇宙感" |
| 모드 전환 | 深色 3D 星系 ↔ 浅色 2D 学习模式切换流畅 |

### 3.2 Phase 1+：词库数据

| 指标 | 目标 |
|------|------|
| 词卡完整度 | 含 meaning / scenario / etymology / mnemonic / examples / relations |
| AI 输出格式 | 100% 通过 JSON Schema 校验，无格式错误 |
| 한자어 标注 | TOPIK I 词汇中 한자어 识别准确率 > 90%（人工抽检）|
| 被动/使动形式 | 仅输出经词典验证的形式，不输出无把握的推断 |

### 3.3 核心功能验收

| 功能 | 验收标准 |
|------|---------|
| 词卡详情 | 点击节点 200ms 内打开详情面板 |
| 搜索定位 | 输入韩语/中文/罗马字，500ms 内高亮目标节点并飞行过去 |
| TTS 播放 | 首次生成后缓存，第二次播放无需请求 API，延迟 < 100ms |
| 한자어 模式 | 切换汉字词根金色连线层，动画流畅，无帧率下降 |
| 변형轮 | 동사/형용사 变形轮展开完整，피동/사동 标注"词典验证"标志 |
| 用户编辑 | 可修改 AI 生成的词卡字段，修改后本地持久化 |

## 四、工程约束（精简版）

以下约束必须在第一行代码前就确立：

```
1. React 管场景结构，Three.js 管大批量对象
   错误：{words.map(w => <WordNode key={w.id} />)}
   正确：<InstancedMesh count={words.length} /> + useEffect 写入矩阵

2. 3D 坐标来自数据库，不实时计算
   启动时读取 map_x / map_y / map_z，无 force layout 运行时调用

3. 标签按 LOD 层级显示，不是每个节点都挂标签

4. AI 输出必须 JSON Schema 校验后才写入数据库

5. 피동/사동 形式必须词典验证，规则引擎只生成基础语尾变化

6. hanja 字段来自词典数据，不来自 hangul→hanja 算法转换
```

## 五、Claude Code 执行策略

### 5.1 基本原则

**不要一次性生成完整系统。** 每次给 Claude Code 一个明确边界的任务。

### 5.2 任务拆分顺序

```
Step 1  项目骨架
        Vite + React + TypeScript + Tailwind + shadcn/ui
        只搭结构，不写业务

Step 2  3D 星系原型（mock 数据）
        Three.js + R3F + InstancedMesh + Stars + CameraControls
        Bloom + DOF + CSS2DRenderer 标签
        不接数据库，只验证视觉

Step 3  SQLite 词库
        Schema 建表 + FastAPI 基础路由
        /api/map/nodes 返回 mock 结构的真实数据

Step 4  真实词库接入
        替换 mock 数据为 SQLite
        聚类坐标、汉字词根连线

Step 5  词卡 + TTS + 변형轮
        AI 词卡生成 Pipeline
        Edge TTS 缓存
        변형 规则引擎

Step 6  TOPIK / PDF（后期）
```

### 5.3 每次 Prompt 写法

**不好的写法：**
```
帮我做一个完整的韩语学习宇宙星图系统
```

**好的写法：**
```
只实现 StarMap.tsx 组件。
输入：words 数组（含 id/map_x/map_y/map_z/color/label）
要求：
1. Three.js + React Three Fiber
2. InstancedMesh 渲染节点（发光球体）
3. CameraControls 缩放/旋转
4. 点击节点后相机平滑飞行过去
5. CSS2DRenderer 标签，只在相机距离 < 80 时显示
不要实现数据库，不要实现词卡面板，不要实现搜索
```

### 5.4 推荐配合的 Skills

| Skill | 用途 |

| `frontend-design` | 复刻深色宇宙 UI、词卡详情面板、底部工具栏 |
