**日期：** 2026-05-20

## 一、产品定位

### 目标用户
淘宝 / 拼多多 / 抖音小店卖家（个人卖家、中小型店铺）

### 核心价值主张
> 商品图不用拍、不用修、不用等——上传商品，批量生成主图，直接上架

### 量化卖点（用户付钱的真实理由）
- **省拍摄成本：** 一张白底图 → 生成多种场景图，替代摄影棚
- **省修图成本：** 自动换背景、场景合成，无需 PS
- **提速上新：** 批量处理多个 SKU，1 小时完成过去一天的工作量
- **降测图风险：** 同款生成多版主图，低成本 A/B 测试点击率

### 附加卖点
- 无需翻墙，国内直接访问
- 无需懂提示词，中文关键词或选模板即可出图

## 二、核心功能

### 三种生成入口

| 入口 | 适合用户 | 操作方式 |
|------|----------|----------|
| 场景模板 | 新手用户 | 选平台 → 选风格 → 上传商品图 → 一键生成 |
| 关键词输入 | 有想法但不懂提示词 | 输入中文关键词 → 系统自动扩展为 Prompt → 生成 |
| 自定义 Prompt | 高级用户 | 直接输入提示词，完全控制 |

### 模板库（MVP 阶段：10 个核心模板）
- 淘宝主图（800×800）
- 拼多多主图（800×800）
- 抖音小店封面（1:1 / 3:4）
- 白底图（通用）
- 室内陈列场景
- 户外生活场景
- 简约文艺风
- 科技感背景
- 节日促销风格
- 高端奢侈品风格

### 批量生成（第二阶段）
- 同一商品一次生成多种风格
- 批量上传多个 SKU，队列处理
- 批量导出为 ZIP

### 历史记录
- 查看所有生成记录
- 一键下载单张或打包下载
- **免费用户图片保存 7 天，付费用户保存 30 天**
- 过期自动删除，用户可提前手动删除

## 三、技术架构

### 整体架构

```
用户（国内浏览器）
    │
    ▼
前端 Next.js + Tailwind CSS
    │  Vercel 全球 CDN 部署
    ▼
后端 Python FastAPI
    │  AWS 新加坡 / 阿里云香港
    ├─→ OpenAI GPT-4o API（关键词扩展为英文 Prompt）
    ├─→ OpenAI GPT-Image-2 API（图片生成，token 计费）
    ├─→ PostgreSQL（用户、积分、订单、生成历史）
    ├─→ Redis（限流、任务队列、会话缓存）
    └─→ Cloudflare R2（生成图片存储，按需 CDN 分发）

支付（早期）：虎皮椒 / PayJS（待上线前核实限额与费率）
支付（中期）：官方微信支付商户号 + 支付宝（注册个体户后升级）
```

### 关键数据流

**场景模板生成：**
1. 用户选模板 + 上传商品图
2. 后端预估本次消耗积分数，前端展示确认
3. 后端取预设 Prompt 模板，填入变量
4. 调用 GPT-Image-2 API（image edit 模式，参考商品图）
5. 返回图片 URL，存入 Cloudflare R2
6. 扣减用户积分，写入生成历史；失败则退还积分

**关键词生成：**
1. 用户输入中文关键词
2. 后端调用 GPT-4o，将关键词扩展为高质量英文 Prompt
3. 预估积分 → 用户确认 → 调用 GPT-Image-2 生成
4. 余下流程同上

### 成本风控（必做三件事）

1. **生成前预估积分：** 根据尺寸、质量档位、是否上传参考图，在生成前计算预估 token 消耗，显示给用户并扣除对应积分，失败退还
2. **每日免费次数限制：** 免费用户每日上限 1 次生成（赠送 3 张用完即止）
3. **单用户最大消耗限额：** 后台可配置每用户单日/单月最大消耗上限，防止异常刷量

> **注意：** GPT-Image-2 按 token 计费（image input $8/1M tokens，image output $30/1M tokens，text input $5/1M tokens）。实际成本受尺寸、质量档位、是否上传参考图、失败重试等多因素影响，**不能按固定单价核算**。上线初期需要跑真实数据，持续监控 API 账单，再反向校准积分定价。

### 数据库核心表

```
users          — id, email, phone, credit_balance, created_at, tier
credit_orders  — id, user_id, amount_cny, credits, status, paid_at
generations    — id, user_id, mode, prompt, image_url, credits_used, tokens_used, status, created_at
templates      — id, name, category, platform, prompt_template, thumbnail_url
```

### 服务器成本估算

| 项目 | 单价 | 备注 |
|------|------|------|
| GPT-Image-2 API | token 计费，需实测 | 受尺寸、质量、输入图影响，上线后监控 |
| GPT-4o（Prompt扩展） | ~$0.001/次 | 可忽略 |
| Cloudflare R2 存储 | $0.015/GB/月 | 基本免费额度内 |
| AWS 新加坡服务器 | ~$20/月（t3.small） | 起步够用 |
| 短信验证码 | ~¥0.05/条 | 阿里云/腾讯云 |

## 六、法务 & 合规

### 上线前必须完成的 5 个页面/条款

1. **服务条款**
2. **隐私政策**
3. **退款规则**（明确不可退、可退条件，减少纠纷）
4. **AI 生成内容免责声明**（见下方具体条款）
5. **用户上传素材授权说明**

**AI 免责声明必须包含：**
- 用户必须拥有上传商品图的合法使用权
- 用户不得上传侵权品牌、明星肖像、盗版图片
- 生成图片仅作为设计辅助，用户上架前需自行审核
- 平台不承诺生成图片一定提升点击率或销量
- 禁止生成虚假宣传、夸大功效、仿冒品牌、误导消费者的图片
- 输出内容可能不唯一，其他用户可能得到类似输出（遵循 OpenAI 使用条款）

使用虎皮椒 / PAYJS 作为个人开发者早期收款方案。上线前需核实：

## 七、冷启动获客策略

**核心原则：发结果，不发功能。**

不要发：「我做了一个 AI 商品图工具」  
要发：「我用一张普通白底图，生成了 8 张拼多多主图，看看哪张更像能提高点击率」

### 具体内容方向

1. **对比图：** 普通商品图 vs AI 场景图，视觉冲击最强
2. **一商品多图：** 同一商品生成 10 张不同风格主图
3. **平台专项：** 拼多多低价促销风、淘宝简约白底、抖音小店封面
4. **失败案例：** 哪些商品图 AI 生成效果差（真实感，建立信任）
5. **成本对比：** 拍摄+修图 vs AI 生成的时间/金钱对比

### 渠道优先级

1. 拼多多/淘宝/抖音卖家 QQ 群、微信群（直接触达目标用户）
2. 抖音/快手/小红书短视频（对比图效果最适合短视频）
3. 闲鱼（可以直接测试用户是否愿意付钱）
4. SEO（「AI商品图生成」「免费商品主图工具」等关键词，长期积累）
